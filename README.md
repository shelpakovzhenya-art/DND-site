# DND site

Рабочее имя интерфейса: **Aisol Pathfinder**.

Редактируемый архив Pathfinder по публичным данным Pathfinder Family: классы, народы, навыки, черты, трейты, заклинания, домены, архетипы, волшебные предметы, оружие, броня, снаряжение, существа, персонажи, заметки и кубики.

## Возможности

- 6773 локальные seed-записи из публичных API Pathfinder Family.
- Полные detail-данные для классов, народов, навыков, черт, заклинаний, волшебных предметов, существ, доменов и архетипов.
- Оружие, броня, трейты и обычное снаряжение подтянуты из list API, где сайт хранит полные карточки.
- Поиск по архиву и фильтры по разделам.
- Редактор записей: можно править существующие записи и добавлять homebrew.
- Персонажи: создание и редактирование листов с характеристиками.
- Кубики: физический 3D-бросок через Three.js/Cannon ES для `d2`, `d4`, `d6`, `d8`, `d10`, `d12`, `d20`, `d100` и формул вроде `2d6+3`; fallback остается для неподдерживаемых костей вроде `d3`.
- Быстрые заметки, избранное, недавние просмотры и генераторы.

## Запуск

```bash
npm install
npm run dev
```

Откройте локальный URL, который покажет Vite, обычно `http://localhost:5173/`.

## Сборка

```bash
npm run build
```

Готовые файлы появятся в `dist/`.

## Обновление данных

```bash
npm run seed
```

Если Node не доверяет локальной цепочке сертификатов, запустите:

```bash
NODE_OPTIONS=--use-system-ca npm run seed
```

На Windows PowerShell:

```powershell
$env:NODE_OPTIONS='--use-system-ca'; npm run seed
```

## Env

Скопируйте `.env.example` в `.env` при необходимости.

```env
VITE_SEED_PATH=/data/pathfinder-seed.json
VITE_APP_NAME=Aisol Pathfinder
VITE_SOURCE_URL=https://pathfinder.family
```

`.env` не коммитится. Для публичного репозитория храните только безопасный `.env.example`.

## Данные и редактирование

Seed лежит в `public/data/pathfinder-seed.json`. Это большой файл с полными API-ответами, поэтому после первого открытия сайт использует seed как базу, а в `localStorage` браузера сохраняет только пользовательские правки и новые записи:

- архивные записи;
- персонажи;
- избранное;
- история бросков;
- быстрые заметки.

Для полного серверного зеркала следующим шагом можно подключить backend-хранилище и заменить `localStorage` на API.

3D-кубики используют ассеты из `public/assets/dice-box-threejs/`; их нужно деплоить вместе с приложением.
